# Casus VipLive Lab
Bij deze de casus voor de technische opdracht. Besteed hier niet te veel tijd aan, maximaal twee dagdelen. Je uitwerking hoeft geen volledige productiewaardige applicatie te zijn, maar dient als stof voor een tweede gesprek.

Besef goed dat je in de gegeven tijd je het nooit helemaal af kan krijgen, dus bepaal voor jezelf waarvan jij vindt dat je oplossing het in ieder geval moet bieden.

We zijn geïnteresseerd in hoe je problemen analyseert, afwegingen en keuzes maakt, zowel qua functionaliteiten die je bouwt als qua technieken die je gebruikt.

## API-implementatie
Schrijf een hele simpele planningsapplicatie die als back-end wordt aangeboden, in de vorm van een REST API.

We gaan ervan uit dat er één agenda is om afspraken in weg te schrijven. Afspraken mogen elkaar niet overlappen.

De API moet ondersteunen:
- Het inplannen van een nieuwe afspraak
- Het opvragen van het eerstvolgende vrije moment waarop een afspraak ingepland zou kunnen worden.

Het is aan jou om te bedenken 
 - welke object-structuur je wilt gebruiken voor de agenda en de afspraken
 - welke input en output je API biedt en verwacht 
 - hoe je eventueel omgaat met foutmeldingen

## Technische hulp
Als implementatie-taal en framework zien we graag Kotlin en Spring Boot. Voor het schrijven van Kotlin raden we je sterk aan om IntelliJ IDEA te gebruiken.

Om je op weg te helpen is hier een eenvoudig projectje met wat variaties op een Hello World API. Voel je vrij om deze volledig te verbouwen of zelf opnieuw te beginnen.

Mocht je technische issues hebben met dit project aan de praat krijgen, probeer dan zelf een applicatie te bouwen. Wil dat ook niet door technische problemen, wijk dan liever uit naar een vertrouwd BE framework en taal en besteed je tijd aan het API-ontwerp.
